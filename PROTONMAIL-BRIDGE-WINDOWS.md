## Step 2: Configure Himalaya for Windows

### Install Himalaya on Windows

**Option A: Using Scoop (Recommended)**

```powershell
# Install Scoop (if not installed)
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
Invoke-RestMethod -Uri https://get.scoop.sh | Invoke-Expression

# Install Himalaya
scoop install himalaya
```

**Option B: Using Chocolatey**

```powershell
# Install Chocolatey (if not installed)
Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))

# Install Himalaya
choco install himalaya
```

**Option C: Download Binary Directly**

```powershell
# Download latest release
cd C:\Users\stgda\Downloads
Invoke-WebRequest -Uri "https://github.com/pimalaya/himalaya/releases/latest/download/himalaya-x86_64-windows.zip" -OutFile "himalaya.zip"

# Extract
Expand-Archive -Path "himalaya.zip" -DestinationPath "C:\Users\stgda\himalaya"

# Add to PATH
[Environment]::SetEnvironmentVariable("Path", $env:Path + ";C:\Users\stgda\himalaya", "User")
```

## Step 3: Create Himalaya Configuration (Windows)

### Create Config Directory

```powershell
# Create config folder
New-Item -ItemType Directory -Force -Path "$env:APPDATA\himalaya"
```

### Create Config File

Save this as `$env:APPDATA\himalaya\config.toml`:

```toml
[accounts.proton]
email = "182gandalf@proton.me"
display-name = "FlowCast Bot"
default = true

# IMAP backend (connects to ProtonMail Bridge)
backend.type = "imap"
backend.host = "127.0.0.1"
backend.port = 1143
backend.encryption.type = "none"
backend.login = "182gandalf@proton.me"
backend.auth.type = "password"
# Use the bridge-generated password here
backend.auth.raw = "PASTE-BRIDGE-PASSWORD-HERE"

# SMTP backend
message.send.backend.type = "smtp"
message.send.backend.host = "127.0.0.1"
message.send.backend.port = 1025
message.send.backend.encryption.type = "none"
message.send.backend.login = "182gandalf@proton.me"
message.send.backend.auth.type = "password"
message.send.backend.auth.raw = "PASTE-BRIDGE-PASSWORD-HERE"

# Folder aliases
[accounts.proton.folder.alias]
inbox = "INBOX"
sent = "Sent"
drafts = "Drafts"
trash = "Trash"
spam = "Spam"

# Signature
signature = "---\nBest regards,\nFlowCast Support Team\nsupport@flowcast.space"

# Downloads directory
downloads-dir = "C:\\Users\\stgda\\Downloads\\himalaya-emails"
```

**Replace `PASTE-BRIDGE-PASSWORD-HERE` with the password from ProtonMail Bridge.**

## Step 4: Secure Password Storage (Windows)

### Option A: Windows Credential Manager (Recommended)

```powershell
# Install CredentialManager module
Install-Module -Name CredentialManager -Force -Scope CurrentUser

# Store password securely
$password = Read-Host -AsSecureString "Enter Bridge Password"
New-StoredCredential -Target "himalaya-proton" -UserName "182gandalf@proton.me" -SecurePassword $password -Comment "ProtonMail Bridge password for Himalaya"
```

Then update config.toml:
```toml
backend.auth.cmd = "powershell -Command '(Get-StoredCredential -Target himalaya-proton).GetNetworkCredential().Password'"
message.send.backend.auth.cmd = "powershell -Command '(Get-StoredCredential -Target himalaya-proton).GetNetworkCredential().Password'"
```

### Option B: Environment Variable

```powershell
# Set environment variable (persistent)
[Environment]::SetEnvironmentVariable("PROTON_BRIDGE_PASSWORD", "your-bridge-password", "User")

# Reload environment variables
$env:PROTON_BRIDGE_PASSWORD = [Environment]::GetEnvironmentVariable("PROTON_BRIDGE_PASSWORD", "User")
```

Update config.toml:
```toml
backend.auth.cmd = "powershell -Command 'Write-Output $env:PROTON_BRIDGE_PASSWORD'"
message.send.backend.auth.cmd = "powershell -Command 'Write-Output $env:PROTON_BRIDGE_PASSWORD'"
```

## Troubleshooting

### Issue: "Connection refused"
**Cause:** Bridge not running or wrong port
**Fix:** Make sure ProtonMail Bridge is open and connected

### Issue: "Authentication failed"
**Cause:** Using ProtonMail password instead of bridge password
**Fix:** Use the bridge-generated password (shown in Bridge app after login)

### Issue: Himalaya not found
**Cause:** Not in PATH
**Fix:** Restart PowerShell or add Himalaya directory to PATH

### Issue: Windows Defender blocks Bridge
**Cause:** Security software
**Fix:** Add ProtonMail Bridge to Windows Defender exclusions
