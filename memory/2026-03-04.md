## Long-Term Learning

### Insight: The "Two Fix Rule"

When the same issue resists 2+ fixes at the same abstraction layer (CSS/JS), the problem exists at a **higher layer** (configuration, architecture, API). Continuing to apply same-layer fixes wastes effort and delays resolution.

**Application:**
- UI issue + 2 DOM fixes → Check component props/configuration
- Auth issue + 2 token fixes → Check session management architecture
- Build issue + 2 config fixes → Check dependency tree
