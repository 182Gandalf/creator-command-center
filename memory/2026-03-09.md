## Key Lessons Learned Today

### Clerk Auth Pattern
Always call `await window.Clerk.load()` before ANY `window.Clerk.session.getToken()` call, even in functions where the user is known to be logged in. Missing this causes `TypeError: Cannot read property 'getToken' of null`.

### FastAPI ORM Serialization
Returning SQLAlchemy ORM objects directly can cause silent 500s if:
- Pydantic model lacks `from_attributes = True`  
- Array columns (ARRAY(String)) return NULL for old records
- `Optional[List[str]]` vs `List[str]` matters for NULL handling
**Fix**: Return explicit dict manually rather than relying on ORM → Pydantic auto-conversion.

### AI Timeout Hierarchy
- Cloudflare proxy timeout: 100s (hard limit, returns 524)
- AI client timeout: set to 80s (just under Cloudflare limit)
- Frontend: handle 524 and 504 with "try again" message
- Model choice matters: `kimi-k2.5` (huge, slow) → `moonshot-v1-8k` (fast, capable enough)

### Platform Selector Architecture Decision
Chosen: **Global default + per-idea override**
- Global platform stored in `flowcast_workspace` localStorage
- Per-idea overrides stored in `flowcast_workspace.platform_overrides[ideaId]`
- `getIdeaPlatform(ideaId)` resolves effective platform for any idea
