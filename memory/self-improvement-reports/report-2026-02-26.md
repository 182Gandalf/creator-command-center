## 3. Lessons Learned

### 3.1 Credential Management Principles

1. **Never hardcode credentials** - Environment variables only, no exceptions
2. **No fallback secrets** - Code should fail loudly if env vars missing, not use defaults
3. **Audit before commit** - Check for secrets before every push
4. **Git history is forever** - Once committed, secrets must be rotated

### 3.2 Infrastructure Decision Making

1. **Check account health first** - Before suggesting services, verify account status
2. **Verify pricing/features** - Don't assume free tiers include needed features
3. **Have backup options** - When primary fails, have alternatives ready
4. **Document blockers immediately** - Don't let infrastructure issues pile up

### 3.3 Security-First Development

1. **Use standard libraries** - werkzeug for passwords, python-dotenv for env vars
2. **Security is not a feature** - It's foundational, implement from day one
3. **Prevention > Detection** - Pre-commit hooks catch secrets before they leak
